# Preview Bug Notes

## Τρέχουσα παρατήρηση

Το preview φορτώνει κανονικά την οθόνη `/welcome` και η μετάβαση με το CTA `Προβολή demo εμπειρίας` προς την αρχική buyer οθόνη ολοκληρώθηκε επιτυχώς χωρίς άμεσο crash.

## Ορατά συμπεράσματα

| Σημείο | Παρατήρηση |
|---|---|
| Welcome screen | Κάνει render σωστά στο web preview |
| Navigation προς demo | Η μετάβαση από `/welcome` προς `/` λειτούργησε |
| Buyer dashboard | Εμφανίζεται κανονικά στο preview |
| Πιθανή αιτία | Το σφάλμα ίσως είναι διαλείπον και να ενεργοποιείται σε unmount/remount κύκλους ή σε συγκεκριμένη αλληλεπίδραση |

## Επόμενη υπόθεση ελέγχου

Θα ελεγχθούν διαδοχικές μεταβάσεις μεταξύ welcome, tabs και nested touchable περιοχών, καθώς και πιθανές ασυμβατότητες web rendering σε σύνθετες clickable κάρτες.

## Μετά το refactor nested touchables

Η welcome οθόνη φορτώνει ξανά κανονικά στο preview και η μετάβαση προς το buyer dashboard ολοκληρώνεται χωρίς εμφάνιση του προηγούμενου `removeChild` crash. Στην αρχική buyer οθόνη εμφανίζονται πλέον ξεχωριστά CTAs `Λεπτομέρειες` και `Προσθήκη`, κάτι που επιβεβαιώνει ότι αφαιρέθηκε το nested interactive pattern από τα product cards.

## Έλεγχος ροής εγγραφής μετά τη διόρθωση ρόλου

Η οθόνη `/sign-up` άνοιξε μία φορά με προσωρινό μήνυμα μη διαθεσιμότητας, αλλά η βασική welcome ροή του preview φορτώνει κανονικά. Επόμενο βήμα είναι η μετάβαση από το CTA της welcome προς την οθόνη εγγραφής, ώστε να επιβεβαιωθεί ότι η φόρμα ανοίγει σωστά μέσα από τη φυσιολογική πλοήγηση της εφαρμογής.

Η μετάβαση από τη welcome οθόνη στη φόρμα δημιουργίας λογαριασμού λειτουργεί κανονικά. Η επιλογή ρόλου δοκιμάστηκε χειροκίνητα στο preview και η εναλλαγή από «Αγοραστής» σε «Προμηθευτής» ενημερώνει σωστά τόσο το οπτικό state όσο και το κείμενο του primary CTA.
