# Horeca Source — Οδηγός δοκιμής (testers)

Καλωσόρισες. Αυτή η σελίδα σου δίνει ό,τι χρειάζεσαι για να δοκιμάσεις το Horeca Source σε ~15 λεπτά: πώς μπαίνεις, τι να δοκιμάσεις σαν buyer, τι σαν supplier, πώς να στείλεις feedback.

## 2 · Σενάρια buyer (5 λεπτά)

Αφού κάνεις sign-in ως `buyer@horeca.demo` βλέπεις 5 tabs στο κάτω μέρος: **Αρχική · Προμηθευτές · Παραγγελίες · Αγαπημένα · Λογαριασμός**.

- [ ] **Αρχική**: χαιρετισμός με το όνομα, KPI strip (παραγγελίες/μήνα, έξοδα), «Γρήγορες ενέργειες», πρόσφατες παραγγελίες.
- [ ] **Προμηθευτές**: τα filter chips («Όλες», «Καφές», «Πρώτες Ύλες», …) αλλάζουν τη λίστα. Πάτα π.χ. **«Καφές»** → μένουν μόνο οι καφέ προμηθευτές. Πάτα **«Όλες»** για reset. Η αναζήτηση πάνω φιλτράρει με όνομα/κατηγορία/τοποθεσία.
- [ ] **Προφίλ προμηθευτή**: πάτα μια κάρτα. Δείχνει avatar, verified badge, rating, tagline, 3-stat grid (Κατηγορία / Παράδοση / MOQ) και τον πλήρη κατάλογό του.
- [ ] **Ροή παραγγελίας**: Προφίλ → προϊόν → «Προσθήκη στο καλάθι» → tab Καλάθι → «Checkout» (mock πληρωμή).
- [ ] **Λογαριασμός → Συνδρομή**: το badge δείχνει «Δωρεάν». Πάτα «Δες τα πλάνα» → **«Αναβάθμιση — €9,90 / μήνα»**. Επιβεβαίωση: το Pro card έχει πλέον «Τρέχον», το tab Παραγγελίες → το button «Εξαγωγή ιστορικού» δεν έχει πλέον lock badge.
- [ ] **Demo-only downgrade**: πίσω στη Συνδρομή → κίτρινο button **«Demo: επιστροφή σε Δωρεάν άμεσα»** → γίνεσαι αμέσως Free (αυτό το button δεν θα υπάρχει στην production έκδοση).

## 3 · Σενάρια supplier (5 λεπτά)

Sign-out → sign-in ως `supplier@horeca.demo`. Τώρα βλέπεις 4 tabs: **Πίνακας · Παραγγελίες · Κατάλογος · Λογαριασμός**.

- [ ] **Πίνακας**: σύνοψη (νέες παραγγελίες, σε επεξεργασία, low stock, έσοδα ημέρας).
- [ ] **Κατάλογος → «+ Νέο προϊόν»**: δοκίμασε να αποθηκεύσεις κενή φόρμα → βλέπεις validation errors. Συμπλήρωσε (π.χ. «Ψωμί ολικής», μονάδα «kg», κατηγορία «Πρώτες Ύλες», τιμή `4.90`) → Αποθήκευση → το προϊόν εμφανίζεται στη λίστα.
- [ ] **Κατάλογος → Επεξεργασία**: πάτα στο εικονίδιο μολυβιού, άλλαξε τιμή, αποθήκευσε → η νέα τιμή είναι ορατή.
- [ ] **Toggle διαθεσιμότητας**: άλλαξε κάποιο προϊόν σε «Περιορισμένο» → το pill αλλάζει αμέσως (optimistic update). Αν κάνεις sign-in ως buyer, θα το δεις και από την πλευρά του κατάλογου.
- [ ] **Διαγραφή προϊόντος**: dialog επιβεβαίωσης → αφαίρεση από τη λίστα.
- [ ] **Παραγγελίες**: φίλτρα στάτους (Νέες / Σε επεξεργασία / Ολοκληρωμένες), άνοιγμα παραγγελίας.

## 4 · Γνωστά όρια του demo

- Η **αναβάθμιση σε Pro** είναι προσομοίωση — δεν υπάρχει πραγματική χρέωση μέχρι την ενσωμάτωση RevenueCat/StoreKit.
- Ο backend server τρέχει σε dev mode σε LAN. Αν δεις «Failed to fetch» στην αρχή, δοκίμασε refresh (pull-to-refresh σε κάθε tab) ή ειδοποίησέ μας.
- **Push notifications** δεν υπάρχουν ακόμη.
- Το tab «Αγαπημένα» αποθηκεύει τοπικά (θα γίνει cloud-synced στο S5).

## 5 · Πώς αναφέρεις bug

Στείλε στο email **[feedback@horeca-source.gr]** ή ανοίξε issue στο GitHub ([link θα σου δοθεί]). Για να βοηθήσεις να το αναπαράγουμε γρήγορα, βάλε:

1. **Screenshot ή screen recording** (iPhone: Shake → «Screenshot» ή τα πλήκτρα power + volume-up).
2. **Βήματα**: «Μπήκα ως buyer → …→ περίμενα Α, είδα Β».
3. **iOS version** (Ρυθμίσεις → Γενικά → Πληροφορίες).
4. **Μοντέλο συσκευής** (π.χ. iPhone 14 Pro).
5. **Ώρα** που συνέβη (για να το βρούμε στα server logs).

---

## 6 · Tips για δοκιμή

- **Reset δεδομένων**: Ειδοποίησέ μας — θα τρέξουμε `pnpm db:setup` για να επαναφέρουμε τα seeded demo data.
- **Αλλαγή dev server IP** (μόνο για native Xcode builds όταν αλλάζει δίκτυο): shake → «Configure Bundler» → βάλε τη νέα IP:port.
- **Xcode rebuild αν εξαφανιστούν αλλαγές**: Product → Clean Build Folder (`Shift+Cmd+K`) → Product → Run (`Cmd+R`).
