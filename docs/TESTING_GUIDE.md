# Εγχειρίδιο testers — Horeca Source (MVP)

Ένα αρχείο για κάθε βοηθό, πελάτη demo ή beta tester: πώς μπαίνεις, τι να ελέγξεις, πού να αναφέρεις προβλήματα.

---

## Τι πρέπει να κάνεις εσύ (developer με φυσικό iPhone)

Δεν είναι κώδικας που γράφεται στο repo — είναι **χειροκίνητοι έλεγχοι** που κάνεις **εσύ** στο Mac και στο τηλέφωνό σου ώστε να επιβεβαιώσεις ότι όλα δουλεύουν όπως στη συσκευή σου.

### Α. Κάθε που δουλεύεις με Xcode + iPhone

1. Στο **Τερματικό**: `cd` στο φάκελο του project και **`pnpm dev`** — το αφήνεις να τρέχει (API + Metro).
2. Στο **Xcode**: άνοιξε το workspace του iOS του project και πάτα **Run** (▶) στο συνδεδεμένο iPhone σου.
3. Αν η εφαρμογή λέει ότι λείπει API ή κάνει timeout: έλεγξε ότι Mac και iPhone είναι στο **ίδιο Wi‑Fi** και ότι στο **`ios/.xcode.env.local`** έχεις σωστή **LAN IP του Mac + την ίδια πόρτα** που δείχνει το log του `pnpm dev` (λεπτομέρειες στην επόμενη ενότητα **«1. Πρόσβαση»**).

### B. Έλεγχος συνδρομής από την αρχή ως το τέλος (buyer demo — μία ολοκληρωμένη φορά)

Στόχος: να δεις ότι **Δωρεάν → Pro (mock) → πάλι Δωρεάν** δουλεύει στο κινητό και ότι το κουμπί **«Εξαγωγή ιστορικού»** κλειδώνει / ξεκλειδώνει ανάλογα με το πλάνο.

Πριν ξεκινήσεις (προτείνεται ώστε να μη «κολλάς» σε παλιό bundle):

1. **Xcode → Product → Clean Build Folder** (`Shift` + `⌘` + `K`).
2. **Product → Run** (`⌘` + `R`).

Μετά στο **iPhone**:

1. Σύνδεση ως **`buyer@horeca.demo`** / **`demo1234`**.
2. Πήγαινε στο tab **Λογαριασμός**.
3. Στην κάρτα **Συνδρομή**, δες ότι το badge είναι **Δωρεάν** · πάτησε **«Δες τα πλάνα»** → πρέπει να ανοίξει η οθόνη συνδρομής.
4. Στην οθόνη συνδρομής πάτησε **«Αναβάθμιση — €… / μήνα»** (ή το ετήσιο αν το δοκιμάζεις έτσι). Μετά την επιτυχία, η κατάσταση πρέπει να δείχνει **Pro**.
5. Γύρισε πίσω στο **Λογαριασμός**: η κάρτα Συνδρομή πρέπει να λέει κάτι σαν **«Διαχείριση συνδρομής»** και το badge **Pro**.
6. Πήγαινε στο tab **Παραγγελίες**. Το κουμπί **«Εξαγωγή ιστορικού»** πρέπει να είναι **χωρίς** λουκέτο Pro (να επιτρέπεται η ενέργεια ή να μη σε σταματάει το paywall).
7. Πήγαινε πάλι **Λογαριασμός → Συνδρομή** (μέσω της κάρτας). Κάτω πρέπει να υπάρχει το κουμπί demo **«Demo: επιστροφή σε Δωρεάν άμεσα»** — πάτησέ το, επιβεβαίωσε στο alert.
8. Επιστροφή στο tab **Παραγγελίες**: το **«Εξαγωγή ιστορικού»** πρέπει να είναι **ξανά κλειδωμένο** για Δωρεάν (με ένδειξη ότι χρειάζεται Pro). Πάτησέ το και δες ότι εμφανίζεται μήνυμα paywall με επιλογή να πας στα πλάνα.

Αν κάτι από τα παραπάνω δεν συμβαίνει, το πρόβλημα είναι ρύθμιση συσκευής/build ή regression — όχι ότι «λείπει §1.3 από ένα αρχείο».

## 1. Πρόσβαση

### Εγκατάσταση / dev

- Τοπικά: τρέξε `pnpm dev` (platform API + Metro). Ακολούθησε τις οδηγίες του project για port & env (`.env` / `.env.horecaapp` αν υπάρχει).
- **Physical iPhone + Xcode:** αν το τηλέφωνο πρέπει να μιλήσει στο API του Mac μέσω Wi‑Fi LAN, αντίγραψε το `ios/.xcode.env.local.example` σε **`ios/.xcode.env.local`** και βάλε `EXPO_PUBLIC_API_BASE_URL=http://<LAN-IP-Mac>:<πόρτα-πλατφόρμας>` (ίδια πόρτα με το log του `pnpm dev`). Μετά από αλλαγή env: Xcode **Clean Build Folder** → **Run**. Ορισμός Team / Signing για το scheme από το Xcode.
- **TestFlight / App Store** (όταν είναι διαθέσιμα): εγκατάσταση από το link ή QR που σου στέλνει ο διαχειριστής.

## 2. Σενάρια δοκιμής — Buyer

- [ ] **Login** → εμφανίζονται **5 tabs**: Αρχική, Προμηθευτές, Παραγγελίες, Αγαπημένα, Λογαριασμός.
- [ ] **Αρχική**: μήνυμα welcome/ρόλος, γρήγορες ενέργειες (μεταξύ άλλων **Έξοδα**), **KPI** (σε εξέλιξη / καθ’ οδόν / προς έλεγχο), λίστα **πρόσφατες παραγγελίες** (αν υπάρχουν· στο **δωρεάν** πλάνο ισχύει παράθυρο **τελευταίων 30 ημερών** όπως και στο tab Παραγγελίες).
- [ ] **Ενεργό κατάστημα (αν υπάρχουν πολλαπλές τοποθεσίες):** αν εμφανίζεται chip row «Ενεργό κατάστημα», δες `/locations` (λίστα + δημιουργία)· οι παραγγελίες στην Αρχική/Παραγγελίες φιλτράρουν ανά επιλεγμένη τοποθεσία όταν είναι σχετικό.
- [ ] **Προμηθευτές** → επιλογή chip κατηγορίας (π.χ. **Καφές**) → ενημερώνεται η λίστα.
- [ ] Άνοιγμα **προφίλ προμηθευτή** → καρδιά **αγαπημένου** (free: έως **3** αποθηκευμένοι, paywall στο 4ο αν δεν είσαι Pro).
- [ ] **Κατάλογος** / προϊόν → **προσθήκη στο καλάθι** → καλάθι ενημερώνεται (συγχρονίζεται με τον **server** μετά το login).
- [ ] **Checkout** → ολοκλήρωση παραγγελίας (mock billing). Πολλοί προμηθευτές → πολλαπλές παραγγελίες.
- [ ] **Παραγγελίες**: badge **μηνιαίου ορίου** (δωρεάν **10**/μήνα)· κάτω από τη λίστα, αν υπάρχουν παραγγελίες **πιο παλιές** από τις πρώτες 30 ημέρες, εμφανίζεται **paywall για πλήρες ιστορικό (Pro)**.
- [ ] **Αγαπημένα**: λίστα από τον server· badge **`n / 3`** (δωρεάν) ή **`n αγαπημένα`** (Pro).
- [ ] **Έξοδα:** από την Αρχική (γρήγορη ενέργεια) ή **Λογαριασμός → Έξοδα & σύγκριση προμηθευτών** → η οθόνη φορτώνει σύνοψη/γραφικές αναφορές (δωρεάν: όπως στο copy του app για εύρος ιστορικού· με **Pro** ευρύτερα εύρη όπου εφαρμόζεται και εξαγωγή αν είναι ξεκλείδωτη).
- [ ] **Λογαριασμός** → **Συνδρομή** → προσωρινή αναβάθμιση **Pro** (dev)· επιστροφή → **Εξαγωγή ιστορικού** στη λίστα παραγγελιών **χωρίς** κλείδωμα Pro όταν το plan είναι ενεργό Pro.

## 4. Πώς αναφέρεις ένα bug

1. **GitHub Issues** του repository που σου υποδείχθηκε από την ομάδα **ή** email στο διαχειριστή που σου έδωσε πρόσβαση demo.
2. Συμπεριλάβεις:
   - **Βήματα** αναπαραγωγής (numbered).
   - **Αναμενόμενο** vs **πραγματικό**.
   - **Screenshot** ή screen recording αν βοηθά.
   - **iOS έκδοση**, μοντέλο συσκευής (ή «Expo Go / dev build»).
   - **Ρόλος** (buyer/supplier) και αν χρησιμοποιείς demo accounts.

---

## 5. Γνωστά όρια του demo

- Η **συνδρομή Pro** είναι **προσομοίωση** (dev endpoint)· **όχι** πραγματική χρέωση App Store / κάρτα.
- Το **backend** τρέχει συχνά σε **dev server**· μπορεί να μην είναι πάντα online.
- Οι **push notifications**: το project φέρει `expo-notifications` σε dev/client· μέχρι παραγωγικής διαμόρφωσης (APNs, hosting) για τους testers εννοείται **αραιή ή ανύπαρκτη** παραδότηση σε πραγματική συσκευή μέσω TestFlight αν δεν τις έχει ενεργοποιήσει η ομάδα ρητά.
- **Εξαγωγή ιστορικού** (PDF/CSV) μπορεί να εμφανίζει «σύντομα» ακόμη και όταν το κουμπί είναι ξεκλειδωμένο — το ίδιο ισχύει για άλλα μελλοντικά features στο copy.

## 6. Tip πριν το Xcode build

Αν οι αλλαγές JS δεν φαίνονται στη συσκευή: **Xcode → Product → Clean Build Folder** και ξανά **Run**. Αν η Metro είναι σε μη-default port, βεβαιώσου ότι το dev client συνδέεται στο bundle που περιμένεις — χρησιμοποίησε τα ίδια βήματα που γράφουν στην ενότητα **«Τι πρέπει να κάνεις εσύ»** στην κορυφή του εγχειριδίου (Clean Build πριν τον έλεγχο συνδρομής).
