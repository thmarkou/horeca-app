## 2. Φάση 1.2 — Cart sync στο backend (πολυσυσκευής + διόρθωση διαρροής ανά χρήστη)

Σύμφωνα με τη συμφώνηση «1.2 = persistent cart στον server».

### 2.1 Τεχνική περιγραφή

| Περιοχή | Λεπτομέρεια |
|---------|--------------|
| **Schema** | Πίνακας **`cart_items`**: `userId`, `productId`, `qty`, `updatedAt`, FK με cascade σε `users` και `products`, **unique (`userId`, `productId`)**. |
| **API** (μόνο **buyers**) | `GET /api/me/cart`, `PUT /api/me/cart/items/:productId` (body `{ qty }`, 0–9999· `qty === 0` αφαιρεί), `DELETE /api/me/cart`, `DELETE /api/me/cart/supplier/:supplierId`. Φόρτωση μέσω JOIN σε products/suppliers για **τρέχουσα** τιμή/όνομα. |
| **`lib/cart-sync.ts`** | `syncedAddItem`, `syncedSetItemQty`, `syncedRemoveItem`, `syncedClearBySupplier`, `syncedClear` — local Zustand + write-through προς τον server (`isHydrating` για bootstrap). **`bootstrapCartFromServer()`**: αν ο server έχει γραμμές → **νικά** την τοπική σειρά/περιεχόμενο· αλλιώς αν υπάρχει τοπικά cart → ανέβασμα γραμμών. |
| **`lib/cart-pricing.ts`** | `parseMinimumOrderEur()` για Ελληνικό MOQ κείμενο (€, ευρώ, EUR, κόμμα σε δεκαδικά). |
| **Πλοήγηση** | Στο **`navigateAfterHorecaAuth`** για buyer: `void bootstrapCartFromServer()` πριν `replace('/(tabs)')`. |
| **Έξοδος** | **`signOut()`** καλεί **`clearCartLocal()`** ώστε να μην μένουν προϊόντα πρώτου χρήστη στο ίδιο κινητό. |
| **Οθόνες** | `catalog`, `product-detail`, `cart`, `checkout`, `order-detail` χρησιμοποιούν τα **synced\*** για mutations. |
| **Checkout polish** | Λωρίδα **Προμηθευτές / Είδη / Σύνολο**· προειδοποίηση όταν υποσύνολο < MOQ ανά προμηθευτή (με `useSuppliersListQuery`) χωρίς να μπλοκάρει submit. |
| **Βάση** | Εφαρμοσμένο μέσω **`pnpm db:push`** (πίνακας `cart_items`). |

### 2.2 Εργαλεία & ποιότητα

- **`pnpm check`**, **`pnpm test`** (κατάσταση codebase: **103 tests** που περνούν — βλέπε §5 για τελευταίο έλεγχο τεκμηρίωσης).
- Smoke με `curl` στο **localhost:3010** για cart + role guards (401 / 403).
