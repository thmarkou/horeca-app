# Κατάσταση έργου — 16 Απριλίου 2026

## Περίληψη ημέρας

Σήμερα εστιάσαμε στη **σύνδεση της εφαρμογής (Expo / React Native) με την τοπική πλατφόρμα API (Hono, θύρα 3010)**, στη **βελτίωση μηνυμάτων σφάλματος** και στην **εμπειρία σύνδεσης**. Το **`pnpm dev`** τρέχει κανονικά (πλατφόρμα σε `0.0.0.0:3010`, Metro συχνά σε **8092** αν το 8091 είναι πιασμένο). Παρά τα παραπάνω, η **άμεση πρόσβαση από το iPhone στο API μέσω LAN παραμένει αναξιόπιστη ή αποτυγχάνει** στο περιβάλλον δοκιμών — λεπτομέρειες παρακάτω.

## Αλλαγές στον κώδικα (16/4/2026)

### Δίκτυο / API client (`lib/api/http.ts`)

- Διόρθωση χρήσης σταθεράς στο timeout: το `throw` στο `catch` χρησιμοποιεί πλέον σωστά `getHorecaApiTimeoutMessageGr()` (προηγουμένως ασυνέπεια ονόματος που μπορούσε να χαλάσει το χειρισμό του timeout).
- Ανίχνευση timeout (`AbortError`, μηνύματα τύπου «network request timed out» κ.λπ.) και **διακριτά μηνύματα ανά περιβάλλον**:
  - **`__DEV__`:** οδηγίες για **τοπική ανάπτυξη** (ίδιο Wi-Fi, `pnpm dev`, σωστή IP/πόρτα στο `EXPO_PUBLIC_API_BASE_URL`). Δεν στοχεύουν σε τελικό χρήστη App Store.
  - **Release (π.χ. App Store):** σύντομο μήνυμα για ασταθές δίκτυο και δοκιμή ξανά — **χωρίς** αναφορές σε Mac, `3010` ή μεταβλητές περιβάλλοντος developer-only.

### Οθόνη σύνδεσης (`app/sign-in.tsx`)

- Αντιστοίχιση αγγλικού μηνύματος στο ίδιο ελληνικό κείμενο με το HTTP layer (ώστε να μην εμφανίζεται «Network request timed out» απευθείας).
- **Πεδίο κωδικού:** εναλλαγή εμφάνισης/απόκρυψης (εικονίδιο Material)· προεπιλογή ορατό κείμενο, ώστε να μην φαίνονται μόνο τελείες κατά την πληκτρολόγηση.

## Αποτυχία σύνδεσης (κατάσταση δοκιμών)

### Συμπτώματα

- Στην εφαρμογή: μετά από αναμονή, **λήξη χρόνου αιτήματος** (timeout περίπου 25 s) κατά το login — εμφανίστηκε **ελληνικό** μήνυμα (μετά τις διορθώσεις), όχι ακατέργαστο αγγλικό κείμενο.
- **Στο ίδιο iPhone**, το URL **`http://192.168.100.40:3010`** (και `/health`): **δεν ανοίγει / δεν φορτώνει** σύμφωνα με τις δοκιμές.

### Τι επιβεβαιώθηκε στο Mac (ανάπτυξη)

- Η Wi-Fi διεύθυνση του Mac (`en0`) ήταν **`192.168.100.40`** — ταυτίζεται με το URL που δοκιμάστηκε στο iPhone.
- Η πλατφόρμα **ακούει** σε **`*:3010`** (όχι μόνο σε `127.0.0.1`).
- Από το Mac, αίτημα προς **`http://192.168.100.40:3010/health`** επιστρέφει **`200`** και JSON `{"ok":true,"service":"horeca-platform"}`.
- Το **τείχος προστασίας (Firewall) του macOS** ήταν **ενεργό** — ενδέχεται να **εμποδίζει εισερχόμενες συνδέσεις** από άλλες συσκευές στο LAN προς τη διεργασία Node, ακόμα κι όταν η πρόσβαση από το ίδιο Mac φαίνεται εντάξει.

### Πιθανά αίτια (μόνο για τοπική δοκιμή, όχι για App Store)

1. **Firewall / εισερχόμενες** στο Mac για `node` (ή το εργαλείο που τρέχει την πλατφόρμα).
2. **Διαφορετικό subnet** στο iPhone (π.χ. άλλο Wi-Fi, hotspot, **Guest Wi-Fi** με απομόνωση clients).
3. **Απομόνωση clients (AP isolation)** στο router — συχνό σε guest δίκτυα.
4. **VPN** στο Mac ή στο iPhone.

**Σημείωση για προϊόν:** Σε **παραγωγή (App Store)** το API θα είναι σε **φιλοξενούμενο endpoint (HTTPS)**· οι τελικοί χρήστες **δεν** ρυθμίζουν firewall στο Mac ούτε LAN IP. Τα παραπάνω αφορούν **μόνο** τοπική ανάπτυξη με φυσική συσκευή.
